## What I build

I design product systems that help humans and AI agents create with shared language, sharper taste, and less friction.

My work sits between product design, component strategy, design tokens, documentation systems, accessibility, and AI-assisted creative workflows. I like systems that are not only beautiful to use, but clear enough to be reasoned over, critiqued, and extended.

## Current focus

- Building **nanaOS**, a personal website and creative operating system for portfolio work, writing, Notion-powered content, and AI-native design system research.
- Developing **nanaOS Design System**, a code-first design system with semantic tokens, accessibility rules, component contracts, previews, and AI-readable generated metadata.
- Exploring **agent workflows for designers**, including critique, drafting, governance, documentation, and structured publishing systems.
- Turning design knowledge into reusable artifacts: skills, prompts, schemas, content engines, case studies, and public writing.

## Principles I care about

- Components should carry intent, usage guidance, constraints, and examples.
- Tokens should describe meaning, not only visual values.
- Documentation should become executable context for agents.
- Accessibility should be a system law, not a late-stage checklist.
- Designers should stay in control of taste, judgment, and final decisions.
